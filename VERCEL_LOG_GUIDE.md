# Vercelのログ確認方法

## 方法1: Vercelダッシュボードから確認（推奨）

### ステップ1: Vercelダッシュボードにアクセス

1. **ブラウザで [https://vercel.com](https://vercel.com) にアクセス**
2. **ログイン**（まだログインしていない場合）

### ステップ2: プロジェクトを選択

1. **ダッシュボードのホーム画面**から、`study-planner-next`プロジェクトをクリック

### ステップ3: デプロイメントを選択

1. **「Deployments」タブ**をクリック
2. 最新のデプロイメント（一番上）をクリック

### ステップ4: ログを表示

1. **「Functions」タブ**をクリック
2. または、**「Runtime Logs」タブ**をクリック
3. または、**「Logs」タブ**をクリック

**ログ画面では以下が確認できます：**
- リクエストのタイムスタンプ
- HTTPメソッド（GET、POSTなど）
- パス（`/api/auth/recent-users`など）
- ステータスコード（200、404、500など）
- コンソールログメッセージ（`console.log()`の出力）
- エラーメッセージ

### ステップ5: リアルタイムログの確認

1. **「Real-time」**または**「Live」**ボタンをクリックして、リアルタイムでログを監視できます
2. ブラウザで`/api/auth/recent-users?migrate=true`にアクセスすると、すぐにログが表示されます

## 方法2: 関数ログから確認

1. プロジェクトページで**「Functions」タブ**をクリック
2. 特定の関数（例：`api/auth/recent-users`）をクリック
3. **「Logs」**セクションでその関数のログを確認

## 方法3: リクエストごとにログを確認

1. **「Deployments」**タブでデプロイメントを選択
2. **「Functions」タブ**をクリック
3. `/api/auth/recent-users`をクリック
4. 個別のリクエストをクリックして詳細を確認

## 確認したいログの例

マイグレーションを実行するために、以下のログを探してください：

```
[MIGRATION DEBUG] Full URL: https://...
[MIGRATION DEBUG] migrateParam: true
[MIGRATION DEBUG] shouldMigrate: true
[MIGRATION] Starting database migration via recent-users endpoint...
[MIGRATION] Migration completed successfully
```

## ログの検索方法

1. **ログ画面の上部に検索ボックス**がある場合、以下のキーワードで検索：
   - `MIGRATION`
   - `migrateParam`
   - `recent-users`

## ヒント

- **時間フィルター**を使って、特定の時間範囲のログを表示できます
- **ログレベル**でフィルタリング（エラーのみ表示など）できます
- **リアルタイムモード**で、新しいリクエストのログを即座に確認できます

## トラブルシューティング

### ログが表示されない場合

1. **デプロイメントが「Ready」になっているか確認**
2. **最新のデプロイメントを確認**（古いデプロイメントのログを見ている可能性）
3. **リクエストを送信したか確認**（アクセスしないとログは生成されません）
4. **ブラウザを更新**して再試行

### 特定の関数のログが見つからない場合

1. **「Functions」タブ**で、関数が正しくビルドされているか確認
2. **デプロイメントログ**で、ビルド時にエラーが出ていないか確認
